## рџ› пёЏ Implementation Guidelines
[[#^toc-guidelines|TOC]]
- **Encoding Safety**: Preserve UTF-8 signatures. Ensure icons, characters, emojis, and unicode symbols are written cleanly without corruption (mojibake).
- **Target Changes Only**: Avoid complete file rewrites. Prefer minor, highly precise surgical patches to retain existing code blocks and comments intact.
- **Edit Source, Not Bundle**: Modify individual module files (`_main.js`, `grid-config.js`, etc.). Never hand-edit `main.js`; it is generated by `build.js`.
- **Obsidian Runtime Boundaries**: Use `require('obsidian')` only in the plugin entry. The bundler skips inlining `obsidian` and `electron` modules.

- Use dashes (`-`) instead of asterisks (`*`) for bullet list items.
- Maintain UPPERCASE.md documents cleanly with alphabetical features lists, updated logs, and checked backlogs.

## вњ’пёЏ Formatting & Syntax Style
[[#^toc-syntax|TOC]]
- **Indentation**: Use 2 spaces per level. No tabs.
- **Braces and Blocks**: Always use braces for control expressions. Avoid single-line `if` without brackets.
- **Module System**: CommonJS вЂ” `require('./module')` and `module.exports = { ... }` or `module.exports = ClassName`.
- **Naming Conventions**:
  - Files: `kebab-case.js` (e.g., `ui-row-select.js`, `main-scanner.js`).
  - Classes: `PascalCase` (e.g., `ProjectGridPlugin`).
  - Functions, variables, object keys: `camelCase`.
  - CSS classes: `projectgrid-kebab-case` prefix on all DOM class names.
  - Global window hooks: `ProjectGrid*` or `window.ProjectGrid*` (e.g., `ProjectGridTriggerFilterUpdate`).
- **Global Function Ordering**: Group requires at top of file by subsystem (see `_main.js` section comments). Export at bottom; assign to `globalThis` when cross-IIFE access is required after bundling.

## рџ›ЎпёЏ Robustness & Error-Handling Frameworks
[[#^toc-errors|TOC]]
- **Primary Paradigm:** Defensive guards with empty `catch` blocks for non-critical filesystem reads; `console.error` for diagnosable failures (timestamps, git extraction).
- **Defensive Coding Checks:** Always validate paths with `fs.existsSync` before `readFileSync`, `statSync`, or destructive writes. Use Obsidian vault APIs (`app.vault.read`, `app.vault.modify`) for note mutations.
- **Logging Integration:** Use `console.log` for plugin lifecycle (`onload`/`onunload`) and `console.error` with `[ProjectGrid]` or module prefix for failures.
- **Inline Comments:** Document the "Why" behind complex logic or architectural workarounds (e.g., DOM load order, IIFE isolation). Do not explain obvious language features.

## рџ“‚ Regions Division Style
[[#^toc-regions|TOC]]
- **File Boundaries**: Every source module wraps content in paired region markers:
  ```text
  // ==========================================
  // START OF FILE: filename.js
  // ==========================================
  ...
  // ==========================================
  // END OF FILE: filename.js
  // ==========================================
  ```
- **Subsystem Grouping in `_main.js`**: Numbered comment sections separate Core Dependencies, Menu Subsystems, Grid Row UI, Metadata Selectors, and Backend Sync Hooks.
- **Bundled Output**: The build strips START/END markers from inlined modules and wraps each in an IIFE assigned to `globalThis`.
